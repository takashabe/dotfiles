# Skill Suggester

## Overview

セッションの決定事項や繰り返し要素をスキル候補として抽出し、提案フォーマットで提示する。
提案は確定せず、合意後に作成・更新へ進む。

## Workflow

1. セッション内の「繰り返し使う手順・方針・判断基準」を抽出する。
2. 既存スキルやガイドラインと重複しないか確認する。
3. スキル候補を1〜3件に絞る。
4. 提案フォーマットで提示し、次のアクション（作成/更新/見送り）を確認する。
5. 合意後に `skill-creator` の手順に進む。

## 提案フォーマット

選択肢を提示する時は、以下の形式を必ず使う。

1. スキル候補名（推奨度：⭐の5段階評価）
   - 理由: 何をスキル化できるか、なぜ価値があるか
   - 想定トリガー: ユーザー発話例を1〜2個
   - 追加すべき資産: `scripts/`, `references/`, `assets/` の要否

## 判断基準

以下のいずれかを満たすもののみ提案する。

1. 今後も繰り返し使う運用ルール・手順
2. 複数タスク/複数ファイルに跨る設計判断
3. 既存のガイドラインに無い、現場固有の知見

以下は除外する。

1. 今回限りの実装詳細
2. 単一ファイルのローカル最適
3. 既存ドキュメントに既にある内容の重複

## 作成/更新の分岐

既存スキルの改善候補がある場合は「更新」を提案し、新規作成が適切なら「作成」を提案する。
合意後に `skill-creator` の手順で初期化・編集・パッケージまで進める。

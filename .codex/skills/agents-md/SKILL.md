# Agents Md

## Overview

AGENTS.md を「汎用的に再利用できる作業ルール」へ整理する。必ず提案→合意→編集の順で進める。

## Workflow

1. 目的の確認
2. 学びの抽出
3. 提案の作成
4. 合意後に編集

## 1. 目的の確認

- 対象ファイルを確認し、既存の方針と重複しないかを把握する。
- 目的は「プロジェクト全体で汎用的に有用なルールのみを残す」こと。

## 2. 学びの抽出

- セッション内の決定事項から、以下に該当するものだけを候補とする。
  - 今後も繰り返し使うルール・手順
  - 仕様や方針の合意事項（例: 常に提案から始める、テストの流儀など）
  - 複数ファイルに影響する設計原則
- 次は除外する。
  - 今回限りの実装詳細
  - 単一ファイルのローカル最適
  - 既存ドキュメント（guidelines/ など）にある内容の重複

## 3. 提案の作成（必須）

- 追加・変更・削除の候補を短い箇条書きで示す。
- 各候補に「理由」を付ける。
- 変更を確定させず、必ず確認を求める。

提案例:

- 追加: 「〜〜」
  - 理由: 〜〜
- 削除: 「〜〜」
  - 理由: 〜〜

## 4. 合意後に編集

- ユーザーの合意を得てから AGENTS.md を編集する。
- 変更は簡潔にし、冗長な説明は避ける。
- 既存のガイドラインへの参照がある場合は重複を避ける。
